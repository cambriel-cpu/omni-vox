### Deployment Issues Fixed
1. SOUL.md mount path wrong → Docker created directory instead of file. Correct: `/mnt/user/appdata/openclaw/config/workspace/SOUL.md`
2. Whisper unreachable from host networking → moved Whisper to host networking too (port 8000)
3. `localhost:18789` no longer reaches OpenClaw → changed to `192.168.68.99:18789` (macvlan via shim)
4. Sessions mount empty at `/mnt/user/appdata/openclaw/agents/` → correct path is `/mnt/user/appdata/openclaw/config/agents/main/sessions/`
5. `docker restart` doesn't re-read `.env` → must stop+rm+run to pick up changes

### Key Lessons
- **Unraid host path mapping is non-obvious:** OpenClaw's `/root/.openclaw/` = host's `/mnt/user/appdata/openclaw/config/` (NOT `/mnt/user/appdata/openclaw/`)
- **Docker creates directories for missing mount sources** — verify paths exist before first `docker run`
- **Moving services from macvlan to host networking** eliminates shim route dependencies (permanent fix > workaround)
- **Whisper restart policy was `no`** — fixed to `unless-stopped` during the move
