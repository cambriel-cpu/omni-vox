**Date:** February 12, 2026  
**Hardware:** AMD Ryzen 9 9950X | 64GB RAM | NVIDIA RTX 4070 Ti Super (16GB VRAM) | Unraid 7.0.0

## Topic 1: Local Whisper Setup on Unraid

### Recommendation: faster-whisper + large-v3-turbo in Docker

**TL;DR:** Run `linuxserver/faster-whisper` Docker container with the `large-v3-turbo` model. It'll use ~2-6GB VRAM (depending on quantization), deliver near-perfect accuracy, and transcribe audio at 10-15x real-time speed on this GPU.

### Whisper Implementation Comparison

| Implementation | Pros | Cons | Best For |
|---|---|---|---|
| **faster-whisper** ⭐ | 4x faster than OpenAI Whisper, lower VRAM via CTranslate2/int8, great Docker support | Slightly more setup than whisper.cpp | **This use case** — best balance of speed, accuracy, Docker integration |
| openai-whisper | Reference implementation, simplest | Slow, high VRAM, CPU-bound bottlenecks | Research/reference only |
| whisper.cpp | Very low VRAM (~2GB for turbo), fast | Less Docker ecosystem, no Python API | Embedded/edge deployments |

### Model Size Guide (RTX 4070 Ti Super — 16GB VRAM)

| Model | Parameters | VRAM (float16) | VRAM (int8) | Speed Factor* | Accuracy |
|---|---|---|---|---|---|
| tiny | 39M | ~1GB | ~0.5GB | 30-50x | Low |
| base | 74M | ~1GB | ~0.5GB | 25-40x | Fair |
| small | 244M | ~2GB | ~1GB | 15-25x | Good |
| medium | 769M | ~5GB | ~3GB | 8-15x | Very Good |
| **large-v3-turbo** ⭐ | 809M | ~6GB | **~2-3GB** | **10-15x** | **Near large-v3** |
| large-v3 | 1550M | ~10-15GB | ~5-6GB | 3-5x | Best |

*Speed factor = how many times faster than real-time on RTX 4070 Ti Super (estimated). A 10x speed factor means 1 minute of audio transcribed in ~6 seconds.

**Winner: `large-v3-turbo` with int8 quantization.** Only ~2-3GB VRAM, 8x faster than large-v3, with minimal accuracy loss. Leaves plenty of VRAM for an LLM.

### How to Deploy on Unraid

**Option 1: LinuxServer Docker Container (Recommended)**

The `linuxserver/faster-whisper` container is available in Unraid Community Apps and provides a Wyoming protocol server. It's well-maintained and GPU-ready.

```yaml
# Docker Compose equivalent
services:
  faster-whisper:
    image: linuxserver/faster-whisper
    container_name: faster-whisper
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=America/New_York
      - WHISPER_MODEL=large-v3-turbo
      - WHISPER_BEAM=1
      - WHISPER_LANG=en
    volumes:
      - /mnt/user/appdata/faster-whisper:/config
    ports:
      - 10300:10300
    runtime: nvidia
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
    restart: unless-stopped
```

**Option 2: Custom faster-whisper API container**

For a REST API (not Wyoming protocol), you can use containers like `fedirz/faster-whisper-server` which exposes an OpenAI-compatible `/v1/audio/transcriptions` endpoint.

```yaml
services:
  whisper-api:
    image: fedirz/faster-whisper-server:latest-cuda
    ports:
      - 8000:8000
    runtime: nvidia
    environment:
      - WHISPER__MODEL=Systran/faster-whisper-large-v3-turbo
      - WHISPER__INFERENCE_DEVICE=cuda
      - WHISPER__COMPUTE_TYPE=int8
```

### Unraid-Specific Considerations

- **NVIDIA GPU Plugin:** Install the "Nvidia-Driver" plugin from Community Apps to pass GPU to containers
- **Container GPU access:** Use `--runtime=nvidia` or the Unraid Docker template GPU passthrough setting
- **Shared GPU:** Multiple containers can share the GPU (Whisper + LLM) — NVIDIA handles memory allocation dynamically
- **Storage:** Models are cached on first run (~1-3GB depending on model). Map a persistent volume for `/config` or model cache

### OpenClaw Integration

OpenClaw already supports audio transcription. From the config at `/app/docs/gateway/configuration-examples.md`:

```json5
tools: {
  media: {
    audio: {
      enabled: true,
      maxBytes: 20971520,
      models: [
        // Use a local Whisper server with OpenAI-compatible API:
        { provider: "openai", model: "whisper-1" },
        // Or CLI fallback:
        // { type: "cli", command: "whisper", args: ["--model", "base", "{{MediaPath}}"] }
      ],
      timeoutSeconds: 120,
    },
  },
},
```

**To use a local Whisper instance with OpenClaw:**

1. Run a faster-whisper server with an OpenAI-compatible API (e.g., `fedirz/faster-whisper-server`)
2. Configure OpenClaw to point to it by setting the OpenAI base URL for transcription to your local server (e.g., `http://YOUR_UNRAID_IP:8000/v1`)
3. Alternatively, use the CLI fallback with a `whisper` binary accessible from OpenClaw's container

The CLI fallback approach (`{ type: "cli", command: "whisper", args: [...] }`) would require the Whisper binary to be installed inside the OpenClaw container or accessible via a shared path.

**Recommended approach:** Run `faster-whisper-server` with OpenAI-compatible API, then configure OpenClaw to use it as the transcription endpoint.

## Topic 2: Running Nemotron 3 Nano Locally

### What is Nemotron 3 Nano?

- **Full name:** NVIDIA Nemotron-3-Nano-30B-A3B
- **Total parameters:** 30B
- **Active parameters per token:** 3.5B (Mixture-of-Experts — only ~10% active at once)
- **Architecture:** Hybrid Mamba-2 + MoE with 23 Mamba/MoE layers + 6 Attention layers; 128 experts + 1 shared per MoE layer, 6 active per token
- **Designed for:** Reasoning and agentic tasks (tool use, coding, math)
- **Languages:** English, German, Spanish, French, Italian, Japanese
- **Released:** December 2025
- **Key benchmarks:** Beats Qwen3-30B-A3B on most reasoning tasks; 3.3x higher throughput than Qwen3-30B-A3B on same hardware

### Will It Fit on 16GB VRAM?

**⚠️ Tight but possible with quantization + CPU offloading.**

| Quantization | Model Size on Disk | VRAM Usage | Fits 16GB? | Notes |
|---|---|---|---|---|
| BF16 (full) | ~60GB | ~60GB | ❌ No | Needs 80GB+ GPU |
| FP8 | ~30GB | ~30GB | ❌ No | Needs 24GB+ |
| Q4_K_M | ~17GB | ~14-18GB | ⚠️ Barely | With expert offloading to CPU, yes |
| Q4_K_S | ~15GB | ~12-16GB | ✅ Tight | Works with small context windows |

**Real-world reports from Reddit:**
- A user with a 4090 (24GB) running Q4_K_XL uses ~14-15GB VRAM with CPU offloading of some expert layers, getting **60 t/s with 256K context**
- On 16GB, you'd need aggressive CPU offloading (`--override-tensor` to push FFN experts to CPU)
- Expect **7-13 t/s** for generation on similar hardware without full GPU fit
- DataCamp recommends **minimum 24GB VRAM** for comfortable usage

**Bottom line:** It *can* run on 16GB with Q4_K quantization + partial CPU offloading, but you'll sacrifice speed and can't run it simultaneously with Whisper at full capacity.

### Best Way to Serve Locally

| Server | Pros | Cons | Recommendation |
|---|---|---|---|
| **Ollama** ⭐ | Dead simple (`ollama run nemotron-3-nano`), auto-quantization, good Unraid support | Less control over quantization | **Best for this use case** |
| llama.cpp | Most control, `--override-tensor` for CPU offloading, fast | More manual setup | Best if you need fine-grained VRAM control |
| vLLM | High throughput, production-grade | Needs more VRAM, complex setup | Overkill for personal use |
| text-generation-inference | HuggingFace ecosystem | Heavy, complex | Not recommended here |

**Ollama is the clear winner** for Unraid. It's available as a Community App, supports GPU passthrough, and Nemotron 3 Nano is already in the Ollama library.

```bash
# Install via Unraid Community Apps, then:
ollama pull nemotron-3-nano:30b-a3b-q4_K_M
ollama serve  # Exposes OpenAI-compatible API on port 11434
```

### OpenClaw Integration with Local LLM

OpenClaw has **native Ollama support**. From the docs:

```json5
{
  agents: {
    defaults: { model: { primary: "ollama/nemotron-3-nano" } },
  },
}
```

Ollama is auto-detected at `http://127.0.0.1:11434/v1`. If running in a separate container on Unraid, use the container's IP or hostname:

```json5
{
  agents: {
    defaults: { model: { primary: "local/nemotron-3-nano" } },
  },
  models: {
    mode: "merge",
    providers: {
      local: {
        baseUrl: "http://YOUR_UNRAID_IP:11434/v1",
        apiKey: "ollama-local",
        api: "openai-completions",
        models: [
          {
            id: "nemotron-3-nano",
            name: "Nemotron 3 Nano 30B",
            reasoning: true,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 32768,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

**⚠️ Important note:** OpenClaw's system prompts and agent loop are token-heavy. Small/slow local models may struggle with the full OpenClaw context. The Reddit thread about Ollama + OpenClaw notes that models smaller than ~16B parameters tend to be too slow for OpenClaw's prompt processing. Nemotron 3 Nano with only 3.5B active parameters may hit this limitation — worth testing.

### Model Comparison for 16GB VRAM

| Model | Total/Active Params | Q4 VRAM | Speed (16GB GPU) | Reasoning | Tool Use | Recommendation |
|---|---|---|---|---|---|---|
| **Nemotron 3 Nano** | 30B/3.5B | ~15-17GB | 7-13 t/s | ⭐⭐⭐ | ⭐⭐⭐ | Great if it fits; needs offloading |
| Qwen2.5-14B | 14B | ~8-9GB | 30-50 t/s | ⭐⭐ | ⭐⭐ | **Best comfort fit for 16GB** |
| Llama 3.1-8B | 8B | ~5GB | 50-80 t/s | ⭐ | ⭐⭐ | Fast but less capable |
| Mistral-7B | 7B | ~4.5GB | 50-80 t/s | ⭐ | ⭐ | Fast, less capable |
| Phi-3-14B | 14B | ~8-9GB | 30-50 t/s | ⭐⭐ | ⭐⭐ | Good alternative to Qwen |
| Qwen3-30B-A3B | 30B/3B | ~15-17GB | 7-13 t/s | ⭐⭐⭐ | ⭐⭐⭐ | Direct competitor to Nemotron |

### Can Whisper + Nemotron Run Simultaneously?

**It depends on quantization and model choices:**

| Combo | VRAM Usage | Feasible? |
|---|---|---|
| faster-whisper turbo (int8) + Nemotron Q4_K_M | ~3GB + ~15GB = **~18GB** | ❌ Exceeds 16GB |
| faster-whisper turbo (int8) + Nemotron Q4_K_M (CPU offload) | ~3GB + ~10GB = **~13GB** | ✅ Yes, with speed penalty |
| faster-whisper turbo (int8) + Qwen2.5-14B Q4 | ~3GB + ~9GB = **~12GB** | ✅ Comfortably |
| faster-whisper turbo (int8) + Llama 3.1-8B Q4 | ~3GB + ~5GB = **~8GB** | ✅ Easily |

**For simultaneous use, Qwen2.5-14B or similar ~14B model is the sweet spot** — leaves room for Whisper and runs at good speed.

If you want Nemotron specifically, use Ollama's automatic model loading/unloading — it loads the model when needed and frees VRAM when idle. Whisper transcription is bursty (only active during voice notes), so in practice they rarely need GPU simultaneously.

## Final Recommendations

### Phase 1: Whisper (Do This First)
1. Install **NVIDIA Driver plugin** on Unraid if not already done
2. Deploy **`linuxserver/faster-whisper`** or **`fedirz/faster-whisper-server`** via Docker
3. Use **`large-v3-turbo`** model with **int8 quantization**
4. Configure OpenClaw to use the local endpoint for audio transcription
5. Test with voice notes — expect transcription in seconds

### Phase 2: Local LLM (Experiment)
1. Install **Ollama** via Unraid Community Apps
2. Try **`nemotron-3-nano:30b-a3b-q4_K_M`** first — see if speed/VRAM is acceptable
3. If too slow or tight on VRAM, fall back to **Qwen2.5-14B** (`ollama pull qwen2.5:14b`)
4. Configure OpenClaw with the Ollama provider config
5. Test as a secondary/fallback model — keep cloud models (Claude/GPT) as primary for complex tasks

### Phase 3: Optimization
- Use Ollama's model scheduling — it loads/unloads models dynamically, so Whisper and LLM can share the GPU
- Consider running Whisper with CPU fallback for when the LLM is active (Ryzen 9 9950X is very capable for Whisper CPU inference)
- Monitor VRAM with `nvidia-smi` to tune the setup
