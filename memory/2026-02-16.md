## Voice Gateway — OpenClaw Integration (Evening Session)

### Problem: Auth Failure
- Voice gateway was calling Anthropic API directly with `ANTHROPIC_API_KEY` env var
- That key is an OpenClaw OAuth token (`sk-ant-oat01-...`), not a raw API key — 401 on direct Anthropic calls
- Chris's Anthropic access is through subscription/OAuth, no raw API key available

### Solution: Route Through OpenClaw Hooks
- Rewired voice gateway to use `/hooks/agent` endpoint (already configured, token in openclaw.json)
- Hooks are async (return 202 with runId, no sync mode, no polling API)
- **Transcript polling approach:** Send via hooks, then poll session transcript JSONL files for assistant response
- Transcript files at `/root/.openclaw/agents/main/sessions/*.jsonl` (NOT in workspace)
- Transcript format is event-based: `{type: "message", message: {role, content}}` — not simple `{role, content}`
- Added `[Voice conversation from Chris...]` prefix so Omni knows to respond conversationally (otherwise gets NO_REPLY)

### Key Fixes
- **Duplicate text bug:** Response included `[[tts:text]]...[[/tts:text]]` block — stripped tags but not content. Fix: remove entire block with `re.DOTALL` regex, then strip remaining `[[ ]]` tags
- **Display vs TTS:** TTS extracts text from tts:text block if present; display removes the whole block
- **Process persistence:** `nohup` required — background `&` jobs die when exec session closes

### Performance
- Whisper STT: ~0.2s
- OpenClaw (Opus via hooks): ~3-6s
- Kokoro TTS: ~0.1-0.2s
- Total: ~3.5-6.5s end-to-end
