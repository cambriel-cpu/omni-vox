## Part 1: Core Dice Mechanics

### Tests vs Checks

The game uses two fundamentally different roll types:

#### Tests (Single Die)
- Roll **one or more D6** individually
- Compare each die result to a **Target Number**
- Result **≥ Target Number** = Success
- Result **< Target Number** = Failure
- **Natural 1 always fails** (regardless of modifiers)
- Can be rolled in batches for efficiency

#### Checks (Two Dice)
- Roll **exactly 2D6** and sum the results
- Compare total to a **Target Number** (usually a Characteristic)
- Total **≤ Target Number** = Success
- Total **> Target Number** = Failure
- **Double 1s = Automatic Success** (regardless of Target)
- **Double 6s = Automatic Failure** (regardless of Target)
- Must be rolled one at a time, never in batches

#### Characteristic Tests
- Use a specific Characteristic as the Target Number
- **Reversed logic**: Result must be **≤** Target Number
- Natural 6 always fails (reversed from normal Tests)
- Used for: Strength Tests, Toughness Tests, Initiative Tests

#### Characteristic Checks
- Used for: Leadership, Cool, Willpower, Intelligence
- Follow standard Check rules (2D6 ≤ Target)
- If Unit has Sergeant/Command Sub-Type, may use their Characteristic

### Modifiers

#### Individual Die Modifiers
- Apply to each die separately (e.g., "+1 to Hit Tests")
- Cannot modify below 1 or above 6
- Applied before comparing to Target
- Addition modifiers apply before subtraction modifiers

#### Total Roll Modifiers
- Apply to the sum of all dice (e.g., "-2 to Check total")
- Cannot reduce total below 0
- For Checks: multiplication applied after addition/subtraction, division last

#### Dice Count Modifiers
- Add or remove dice from the pool before rolling
- Negative modifiers cannot reduce below 1 die
- For Checks: extra dice rolled, then excess discarded (player chooses which)

### Scatter Dice

Used for randomizing Blast positions:

**Scatter Roll** (with Hit icon):
1. Roll Scatter Die + 1D6
2. If **arrow**: Move target point that many inches in arrow direction
3. If **HIT icon**: Small arrow shows direction, but **still moves** the distance rolled

**Indirect Scatter Roll** (Barrage without LoS):
1. Roll Scatter Die + XD6 (where X = Barrage value)
2. If **arrow**: Move target point total inches in arrow direction
3. If **HIT icon**: **No scatter** - target point stays in place

### D3 Results
When rolling D3, use standard D6:
| D6 Roll | D3 Result |
|---------|-----------|
| 1-2 | 1 |
| 3-4 | 2 |
| 5-6 | 3 |

## Part 2: Attack Resolution Flow

Every attack follows this precise sequence:

```
Firepower/Attacks → Hit Test → Wound Test → Saving Throw → Damage Mitigation → Apply Damage
```

### Hits
- Generated by successful Hit Tests
- Carry: Strength, AP, Damage, Special Rules from the weapon
- Can be modified after generation (Breaching, etc.)
- Hits ≠ Wounds (common misconception)

### Wounds
- Generated by successful Wound Tests
- Carry: Strength, AP, Damage, Special Rules from the Hit
- Can be allocated to models by the defending player
- Wounds ≠ Damage (wounds are intermediate state)

### Unsaved Wounds
- Wounds that failed their Saving Throw
- Reduce model's Wounds Characteristic by Damage value
- Excess damage is **lost** (no spillover to other models)

### Penetrating Hits (Vehicles Only)
- Function identically to wounds but for vehicles
- Reduce Hull Points instead of Wounds
- Cannot be allocated to non-Vehicle models

## Part 5: Saving Throws

### Save Types (in order of application)

1. **Armour Save** - Listed on model profile (e.g., 3+)
   - Negated if weapon AP ≤ Armour Save value
   - Example: AP 2 negates 3+ save (2 ≤ 3), but not 2+ save (2 > 2)

2. **Cover Save** - Granted by terrain
   - Light Cover: 6+
   - Medium Cover: 5+
   - Heavy Cover: 4+
   - NOT negated by AP
   - Cannot be used in close combat

3. **Invulnerable Save** - Listed on profile or granted by rules
   - Never negated by AP
   - Can always be taken if available

### Save Selection Rules
- Only **ONE** save may be taken per wound
- Player chooses which available save to use
- Saves do NOT stack
- Always choose the best available save

### Damage Mitigation (Feel No Pain)
- Separate from Saving Throws
- Made **after** failing a Saving Throw
- One Damage Mitigation Test AND one Saving Throw per wound
- Common example: Feel No Pain (5+)

## Part 6: Shooting Phase (11-Step Sequence)

**Prerequisites**: Unit cannot shoot if:
- Rushed (except Light/Cavalry types - Snap Shots only)
- Locked in Combat
- Embarked on Transport

### Shooting Attack Sequence

1. **Select Target**: Choose attacking unit, then target enemy unit
2. **Check Target**: Verify Line of Sight and (for vehicles) weapon facing
3. **Declare Weapons**: State which weapons will fire
4. **Set Fire Groups**: Group weapons by identical BS and weapon profile
5. **Resolve Fire Group**: Roll dice equal to total Firepower
6. **Make Hit Tests**: Compare results to Ranged Hit Test table
7. **Wound Tests**: Roll Wound Tests for all Hits in Fire Group
8. **Assign Wounds**: Defending player allocates wounds to models
9. **Saving Throws**: Roll saves and Damage Mitigation Tests
10. **Next Fire Group**: Repeat steps 5-9 for remaining Fire Groups
11. **Remove Casualties**: Remove models reduced to 0 Wounds

**Critical**: Tactical Statuses gained are applied at **END** of Shooting Phase, not during.

## Part 7: Assault Phase

### Charge Sequence

1. **Declare Target**: Select charging unit and target enemy unit(s)
2. **Check Range/LoS**: Target must be within 12" and visible
3. **Set-up Move**: Move closest model by Set-up Distance (see table)
4. **Volley Attacks**: Both sides may fire (Assault weapons + Snap Shots)
5. **Charge Move**: Roll 2D6, pick highest. Move charging models.

#### Set-up Move Distance Table

| Initiative | Set-up Distance |
|------------|-----------------|
| 1-6 | 1" |
| 7-9 | 2" |
| 10-11 | 3" |
| 12-13 | 4" |
| 14-19 | 5" |
| 20+ | 6" |

### Disordered Charges
Triggered by:
- Disembarking same turn (unless Assault Vehicle)
- Unit has Stunned or Suppressed status

Effects of Disordered Charge:
- No Set-up Move
- No Volley Attacks (but target CAN make Volley Attacks)
- No bonuses from charge-related Special Rules
- Cannot manifest Psychic Powers

### Fight Sub-Phase Sequence

1. **Determine Combat**: Select one ongoing combat
2. **Declare Weapons**: Both sides declare melee weapons
3. **Combat Initiative Steps** (highest to lowest):
   - 3.1 **Pile-in Move**: Move models up to Initiative toward enemy
   - 3.2 **Declare Engaged**: Models in base contact or within 2" may fight
   - 3.3 **Set Strike Groups**: Group attacks by weapon type
   - 3.4 **Hit Tests**: Roll Melee Hit Tests
   - 3.5 **Wound Tests**: Roll Wound Tests for Hits
   - 3.6 **Saving Throws**: Defender rolls saves
   - 3.7 **Remove Casualties**: Remove models at 0 Wounds
4. **Final Pile-in**: All models pile-in by Initiative distance

**Massacre**: If enemy unit is completely destroyed, combat is won. Consolidate immediately.

## Part 8: Challenge System

### Eligibility
Available to non-Routed models with:
- Command Sub-Type
- Champion Sub-Type
- Paragon Type

### Challenge Sequence

1. **Declare Challenge**: Select challenging model (Active Player first)
2. **Face-off**: Select Gambit (Active Player first in Round 1)
3. **Weapon Selection**: Select weapon (Reactive Player first)
4. **Focus Roll**: Winner gains +1 Attack and Challenge Advantage
5. **Strike**: Resolve combat (steps 3.3-3.7 of Fight Sub-Phase)
6. **Challenge Result**: Repeat steps 2-5 or proceed to Glory
7. **Glory Points**: Winner gains Combat Resolution bonus

### Focus Roll Modifiers

| Modifier | Value |
|----------|-------|
| Model's Initiative | +/- I |
| Weapon Initiative Modifier | +/- IM |
| Heavy weapon | -1 |
| Light Sub-Type | +1 |
| Each lost Wound | -1 |
| Shield or Company Standard | +1 |
| Per 5 friendly models in unit | +1 |

### Core Gambits

| Gambit | Effect |
|--------|--------|
| **Seize the Initiative** | Roll extra Focus D6, discard lowest |
| **Flurry of Blows** | +D3 Attacks, but all deal 1 Damage only |
| **Test the Foe** | Gain Challenge Advantage next round |
| **Guard Up** | +1 WS, but only 1 attack. Each enemy miss = +1 Focus next round |
| **Taunt & Bait** | Match opponent's WS & A. +1 Combat Resolution if you win |
| **Grandstand** | Extra Focus D6 (discard highest). +1 A per 5 models in unit |
| **Finishing Blow** | Extra Focus D6 (discard highest). +1 S & D to all hits |

### Disgraced Status (Refusing Challenge)
- Model's WS & LD halved
- Unit's Initiative reduced to 1

### Glory Points
- Winner = model who inflicted most wounds
- Gains Combat Resolution points equal to wounds inflicted
- If enemy slain, gain points equal to their full Wounds Characteristic

## Part 9: Combat Resolution

### Combat Resolution Points

| Source | Points |
|--------|--------|
| Glory Points (from Challenges) | +X |
| Outnumbering enemy | +1 |
| Each enemy Removed as Casualty | +1 |
| Each Vexilla | +1 |
| Each Company Standard | +1 |
| Various Wargear/Special Rules | +/- |

### Combat Aftermath

#### Panic Check (Loser)
The losing side must make a **Panic Check**:
- Roll 2D6 against Leadership
- Apply a **negative modifier** equal to the difference in Combat Resolution Points between winner and loser
- Example: Winner scores 5, Loser scores 2 → Loser rolls at **-3 to Leadership**

**If Panic Check Failed**: The unit must **Fall Back** (no choice)
**If Panic Check Passed**: The unit may choose to **Disengage**

#### Loser Options

- **Fall Back** (mandatory if Panic Check failed): Gain Routed status, move toward deployment zone. Enemy may **Gun Down** (Volley attack)
- **Disengage** (only if Panic Check passed): Normal move away. Enemy may **Pursue** (D6 + Initiative move; contact = new charge)

#### Winner Options

- **Consolidate**: Move up to base Initiative, ignore Difficult Terrain
- **Hold**: Pile-in move. If still in contact, remain in combat

## Part 10: Tactical Statuses

### General Penalties (All Statuses)
- Charges are **Disordered**
- Combat Initiative set to **1**
- Cannot **hold, claim or contest objectives**
- Never benefit from being **Stationary**

### Status Types

| Status | Effect | Caused By |
|--------|--------|-----------|
| **Pinned** | Cannot move, Rush, Charge, Pursue, or Disengage | Failing Cool check from Pinning (X) wound |
| **Suppressed** | All Shooting as Snap Shots | Out of Coherency (Cool), or Suppressive (X) hit |
| **Stunned** | Cannot declare Reactions | Stun (X) hit (Cool check) |
| **Routed** | Must Fall Back each Movement Phase | Panic (X) wound, or >25% casualties (LD check) |

### Routed Unit Behavior
1. Roll D6 + Initiative
2. Move that distance toward own deployment zone edge
3. If contacting board edge, make LD Check:
   - Failed: Unit **removed from play**
   - Passed: Gain **Suppressed** status instead

### Removing Statuses
- **End Phase**: Perform Cool/LD Check or Repair Test (6+) to remove

### Reaction Allotment
- **Base**: 1 Reaction per game turn
- **1500+ pts games**: +1 Reaction
- **3500+ pts games**: +2 Reactions
- **Master of the Legion**: +1 Reaction

### Cannot React If
- Stunned
- Routed
- Locked in Combat
- Already reacted this turn

### Movement Phase Reactions

| Reaction | Trigger | Effect |
|----------|---------|--------|

| **Death or Glory** | Vehicle moves through/stops in contact | One attack; if fail to stop vehicle, model removed |
| **Intercept** | Enemy deploys from Reserves | Snap Shot attacks |

### Shooting Phase Reactions

| Reaction | Trigger | Effect |
|----------|---------|--------|
| **Return Fire** | Enemy shoots at this unit | Shoot back before casualties removed |

## Part 13: Vehicle Rules

### Armour Penetration (replaces Wound Tests)

1. Roll 1D6
2. Add weapon Strength
3. Compare to target Armour Value (AV) of facing hit

| Result | Effect |
|--------|--------|
| Total < AV | No effect |
| Total = AV | **Glancing Hit** |
| Total > AV | **Penetrating Hit** |

### Glancing Hit Table

| D6 | Result | Effect |
|----|--------|--------|
| 1-2 | Impaired Sensors | **Stunned** |
| 3-4 | Broken Motors | **Pinned** |
| 5-6 | Weapons Damaged | **Suppressed** |

**If already has that status**: Lose 1 Hull Point instead (no saves)

### Penetrating Hits
1. Vehicle may make Saving Throws (if available)
2. Vehicle may make Damage Mitigation (if available)
3. If not discarded: Lose HP equal to weapon's Damage

### Vehicle Facings
- **Front**: 0° to 45° each side of center
- **Side**: 45° to 135° each side
- **Rear**: 135° to 180° each side
- Always use facing when attack originates

### Firing Arcs
- **Centreline**: Narrow forward arc (hull weapons)
- **Turret**: 360° arc
- **Sponson**: Side-mounted, 180° to that side

### Transport Rules

**Embarking**:
- Move within 2" of Access Point
- Cannot Embark if Rushed or charged

**Disembarking**:
- Place within 2" of Access Point
- Can move normally after
- Charging after Disembark = Disordered (unless Assault Vehicle)

**Emergency Disembarkation** (Transport destroyed):
- Place within 6" of Access Point
- Roll D6 per model: 1 = wound (no saves)
- Unit gains Pinned status

## Part 14: Model Types

### Infantry
- Standard rules with no modifications
- Can Embark on Transports
- Subject to all terrain effects

### Cavalry
- Fall Back: Initiative + 2D6
- Rush: +3" beyond normal Initiative bonus
- Can Snap Shot when Rushing
- Hammer of Wrath (1) on successful charge

### Walker
- Poisoned triggers only on natural 6
- Can fire all weapons in Shooting Phase
- May make Stomp attacks in combat

### Paragon (Primarchs)
- Hits allocated by **attacking player** (not defender)
- Auto-pass all Leadership/Cool/Willpower Checks
- Poisoned triggers only on natural 6
- Can issue/accept Challenges
- Count as models equal to base Wounds for Combat Resolution

### Automata
- Cannot gain Tactical Statuses
- Poisoned triggers only on natural 6

### Vehicle
- Uses Armour Penetration instead of Wound Tests
- Uses Hull Points instead of Wounds
- Cannot be locked in combat
- Uses Firing Arcs

## Part 16B: Core Mechanics (Units, Coherency, Line of Sight, Measuring)

### Unit Coherency

All models in a unit must maintain coherency:
- Each model must be within **2" horizontally** of at least one other model in the unit
- Must form **one contiguous group** (no isolated models)
- Measured **base-to-base** (or hull if no base)

**Breaking Coherency**:
- If movement would break coherency, unit cannot move that way
- If casualties break coherency, unit must check at end of phase
- Unit out of coherency gains **Suppressed** status (Cool Check to avoid)

### Joining and Leaving Units

Models with **Command**, **Specialist** Sub-Types, or **Paragon** Type can join/leave units:

**Joining a Unit**:
- Must be in Unit Coherency with target unit at **start** of Move Sub-Phase
- Declare joining before any movement
- Joining model gains all Tactical Statuses of the unit
- Unit may not move further that phase after being joined

**Leaving a Unit**:
- Declare when unit is selected to move
- Model must move to end **out of Coherency** with original unit
- Model retains any Tactical Statuses it had
- Cannot leave if: cannot move, Routed, Locked in Combat, or Embarked

**Mixed Faction Units**:
- If unit contains models with different Faction Traits: **-1 LD and CL** to all models
- Loyalists cannot join Traitor units and vice versa

### Line of Sight

**Determining Line of Sight**:
- Draw unbroken straight line from model to target
- If any obstruction (terrain, vehicle hull, etc.) blocks the line = No LoS
- Models in **Base-to-Base contact** always have LoS to each other
- A model always has LoS to itself

**LoS for Attacks**:
- Only models with LoS to target may attack
- To target a unit, at least one model must have LoS to at least one model in target unit
- Having LoS to one model does NOT grant LoS to hidden models in same unit

### Measuring Distances

**General Rules**:
- All distances in **inches**
- Fractions **round UP** to nearest whole inch
- Measured **flat** (parallel to table surface, not at angles)
- Any player may measure any distance at any time

**Measuring Between Models**:
- **With Bases**: Measure closest point on each base (base-to-base)
- **Without Bases** (Vehicles): Measure to hull edge (not weapon barrels or extremities)
- **Markers**: Measure from center of marker/token

**Range**:
- A model is "In Range" if distance ≤ weapon's Range Characteristic
- A model is always in range of itself

### Reserves

**Reserves Roll**: Test with Target Number **3+** per unit in Reserves

**Entering from Reserves**:
1. Select point on Battlefield Edge within your Deployment Zone
2. Place one model at that point
3. Move that model normally (can Rush if desired)
4. Place remaining models, each ending in Unit Coherency
5. Models that cannot end in Coherency are **Removed as Casualties**

**Restrictions**:
- Unit entering from Reserves **cannot move** in Move Sub-Phase (already moved)
- Unit entering from Reserves **can shoot and charge** normally
- Entering from Reserves counts as **having moved** for all rules purposes

### Movement Phase Structure

1. **Reserves Sub-Phase**: Make Reserves rolls, units enter play
2. **Move Sub-Phase**: Move eligible units
3. **Rout Sub-Phase**: Routed units make Fall Back moves

**Move Sub-Phase Procedure**:
1. Select eligible unit
2. Move unit (or remain Stationary)
3. Repeat until all units resolved

**Rushing**:
- Declared when selecting unit to move
- Movement = Movement + Initiative characteristics
- Cannot shoot (except Light/Cavalry Snap Shots)
- Cannot charge same turn

**Moved vs Stationary**:
- **Moved**: At least one model changed position
- **Stationary**: No model changed position (pivoting doesn't count)
- Various rules benefit Stationary units

**Movement Restrictions** (cannot end move):
- In Impassable Terrain
- Within 1" of enemy model
- Within 1" of friendly model Locked in Combat
- Outside Battlefield bounds
- On top of another model

### Buildings

Buildings use special rules:
- Treated as **Vehicles** for targeting and attacking
- **Glancing Hit**: 1 Damage (regardless of weapon Damage)
- **Penetrating Hit**: Damage = weapon's Damage Characteristic
- When destroyed: Replaced by **Difficult Terrain** (5+ Cover Save)
- Embarked unit must **Emergency Disembark** and make **Panic Check**
- **Battlements**: Roof spaces are Medium Area Terrain (5+ Cover), count as Access Points
- No Victory Points scored for destroying Buildings (unless mission specifies)

## Part 17B: Prime Slots System

### What Are Prime Slots?

Some Force Organisation Slots in a Detachment are marked as **Prime Slots** (indicated by a special marker/highlight). Prime Slots retain their Battlefield Role, but if filled with a Unit they offer an **additional benefit**.

### Prime Advantages

For **each Prime Slot filled**, the Controlling Player may select **one Prime Advantage** from the following options:

| Prime Advantage | Effect | Restrictions |
|-----------------|--------|--------------|
| **Master Sergeant** | One Model in the Unit selected to fill the Prime Slot that has the Sergeant Sub-Type gains +1 to its Attacks, Weapon Skill and Leadership Characteristics. The Champion Sub-Type instead increases its Leadership by an additional +1. | May only be selected once per Detachment |
| **Combat Veterans** | All Models in the Unit selected to fill the Prime Slot gain a bonus of +1 to their Leadership, Cool, Intelligence and Willpower Characteristics to a maximum of 10. This includes only Models selected as part of the Unit, not Models that later join the Unit either before or during a Battle. | - |
| **Paragon of Battle** | One Model in the Unit selected to fill the Prime Slot that has the Command Sub-Type gains a bonus of +1 to its Attacks, Weapon Skill and Ballistic Skill Characteristics. | - |
| **Special Assignment** | A Command Slot for which this Prime Advantage is selected may be filled by a High Command Unit, but remains a Command Slot. Moreover, no additional Detachments of any kind may be selected due to this Slot regardless of their Battlefield Role or any Special Rules that the Unit selected to fill the Slot may have. | Command Slots only |
| **Logistical Benefit** | Add one additional Force Organisation Slot to the Detachment that includes the Prime Slot. This Slot may be of any Battlefield Role other than High Command, Command, Warlord or Lord of War. | May only be selected once per Detachment |

### Prime Slot Rules

- If the Unit chosen to fill the Prime Slot includes any Models with the **Unique Sub-Type**, then the only Prime Advantage that may be selected is **Logistical Benefit**
- Prime Advantages that affect Characteristics apply to specified Models in the Unit
- A Unit can only benefit from one Prime Advantage per Prime Slot filled

## Part 17D: Selecting Units for Detachments

### Basic Rules

1. Each Detachment is composed of **Force Organisation Slots**

## Part 19: Psychic Abilities

Psychic abilities represent the eldritch powers wielded by Psykers on the battlefield. These powers are broken down into three distinct types: **Psychic Weapons**, **Psychic Powers**, and **Psychic Reactions**.

### The Psyker Trait

A Model must have the **Psyker Trait** to use any psychic ability. Models without the Psyker Trait:
- Cannot make attacks with Weapons that have the **Psychic Trait**
- Cannot Manifest Psychic Powers or Psychic Reactions
- Cannot use Psychic Gambits

Models with the Psyker Trait do **not** automatically gain any Psychic Weapons or Psychic Powers - these must be granted by a Psychic Discipline or other rule.

---

### Psychic Weapons

Psychic Weapons are Weapons that can only be used by Psykers. They function exactly like other Weapons with the same Characteristics (Range, Firepower, Strength, AP, Damage, etc.).

**Key Rules:**
- Identified by the **Psychic Trait** on the weapon
- Require the Model to have the **Psyker Trait** to use
- Can be used for Shooting Attacks (Ranged) or Combat (Melee Type)
- In close combat, a Psychic Weapon with Melee Type counts for determining if a Model has more than one Weapon
- A Psychic Weapon can **never** be destroyed or removed unless a Rule specifically targets Psychic Weapons
- No additional Tests or Checks are required to attack with a Psychic Weapon (beyond those from Special Rules)
- **Do not count** toward the one-Psychic-Power-per-turn limit

---

### Psychic Powers

Psychic Powers are psychic abilities that achieve effects more akin to Special Rules. They are **not** Weapons and do not inflict Hits, Wounds, or Damage. Psychic Powers are divided into two categories:

| Category | Target | Effect |
|----------|--------|--------|
| **Blessing** | Friendly Unit or Psyker itself | Beneficial effect (buffs, healing, etc.) |
| **Curse** | Enemy Unit | Harmful effect (debuffs, forced movement, etc.) |

#### Usage Restrictions

- A Unit may use **no more than ONE Psychic Power** in the Controlling Player's Turn
- A Unit may use **no more than ONE Psychic Reaction** in the Opposing Player's Turn
- Psychic Weapons and Psychic Gambits do **not** count toward these limits
- If a Model leaves a Unit that has used a Psychic Power, **both** the original Unit and the new Unit are restricted from using further Psychic Powers/Reactions that Turn
- Using a Psychic Power does **not** prevent moving, shooting, charging, or combat (unless the Power states otherwise)
- Models In Reserves, Embarked, or otherwise not on the Battlefield **cannot** use Psychic Powers or Reactions
- Any ongoing effect of a Psychic Power immediately **ends** if the Focus:
  - Embarks on another Model
  - Enters Reserves
  - Is Removed as a Casualty

---

### Manifesting Blessings

To Manifest a Blessing:

1. **Identify the Focus**: The Model using the Psychic Power (must have Psyker Trait)
2. **Select Target**: The Unit to receive the Blessing (friendly Unit or Focus's own Unit)
3. **Make Manifestation Check**: Willpower Check (2D6 ≤ Focus's Willpower)
   - If the Power affects a Unit rather than a specific Model, nominate one Model as the Focus
4. **Apply Effects**:
   - **Passed**: Effects apply to all Target Units
   - **Failed**: No effect, but still counts as having used a Psychic Power

**Note**: Some Blessings may have additional effects that apply even if the Manifestation Check fails - check the specific Power's rules.

---

### Manifesting Curses

To Manifest a Curse:

1. **Identify the Focus**: The Model using the Psychic Power (must have Psyker Trait)
2. **Select Target**: The enemy Unit to be cursed (must be valid per the Power's rules)
3. **Target Makes Resistance Check**: Willpower Check (2D6 ≤ Target's Willpower)
   - Single Model: Use that Model's Willpower
   - Multiple Models: Use the **lowest** Willpower in the Unit, **unless** the Unit includes a Model with Sergeant Sub-Type, Command Sub-Type, or Paragon Type - then the Controlling Player may use one of those Models' Willpower instead
   - Multiple Target Units: Each makes a separate Resistance Check
4. **Apply Effects**:
   - **Resistance Check Passed**: Curse is negated, no effect on that Unit
   - **Resistance Check Failed**: Curse effects apply to that Unit

**Note**: Even if the Resistance Check passes, this still counts as the Focus's Unit having used a Psychic Power for that Turn.

---

### Psychic Reactions

Psychic Reactions are Psychic Powers that can only be used as Reactions during the Opposing Player's Turn. They follow all normal Reaction rules (see Part 12) plus Psychic Power rules.

**Key Rules:**
- Can be Blessings or Curses (follow appropriate rules above)
- Cost Reaction Allotment points as specified
- Some Psychic Reactions may have unique mechanics that don't follow standard Blessing/Curse rules
- Subject to the one-Psychic-Reaction-per-Turn limit

---

### Nullify (Advanced Psychic Reaction)

**Nullify** is the primary method for Psykers to counter enemy Curses.

| Aspect | Detail |
|--------|--------|
| **Trigger** | When a Unit under the Reactive Player's control is targeted by a Psychic Curse |
| **Cost** | 1 Reaction Allotment point |
| **Requirement** | Reacting Model must have the Psyker Trait |

**Process:**

1. **Calculate Power Modifier**: Reacting Model's Willpower **minus** Curse Focus's Willpower
   - This modifier can be **negative** if the enemy Psyker has higher Willpower
2. **Make Willpower Check**: The Reactive Player rolls 2D6, applying the Power Modifier to the result
3. **Resolve**:
   - **Check Passed**: The Psychic Curse **automatically fails** and cannot be re-attempted this Turn
   - **Check Failed**: The Psychic Curse resolves as normal

**Example**: A Librarian (WP 8) attempts to Nullify a Curse from an enemy Psyker (WP 7). The Power Modifier is 8 - 7 = +1. The Librarian rolls 2D6 and adds 1 to the result, needing to roll ≤ 8 to succeed.

---

### Perils of the Warp

When rolling doubles on a Willpower Check made as a **Manifestation Check** or **Resistance Check**, Perils of the Warp triggers.

**Important Notes:**
- **All doubles trigger Perils** - including double 1s (auto-success) and double 6s (auto-fail)
- The Check result still applies (success/failure) in addition to Perils
- Perils affects the Unit that includes the **Focus** of the Psychic Power

#### Perils of the Warp Table

| D6 Result | Effect | Description |
|-----------|--------|-------------|
| **1 or 6** | **Warp Rupture** | The Unit suffers **(13 - Willpower)** wounds. These wounds: ignore Armour Saves, ignore Cover Saves, ignore Damage Mitigation. **Invulnerable Saves may be taken**. Wounds are allocated as if from a Shooting Attack. Use the lowest Willpower in the Unit (or Sergeant/Command/Paragon if present, Controlling Player's choice). |
| **2, 3, 4, or 5** | **Aetheric Feedback** | The Unit gains the **Stunned** Tactical Status. |

#### Perils from Other Sources

When a Special Rule instructs a Player to apply Perils of the Warp (not from a Willpower Check):
1. Roll a single D6
2. Compare to the Perils of the Warp Table
3. Apply the result to the specified Unit

### Psychic Gambits

Psychic Gambits are special Gambits (see Part 8: Challenge System) available to Models with specific Psychic Disciplines. They function as normal Gambits but may require Willpower Checks.

**Key Rules:**
- Do **not** count toward the one-Psychic-Power-per-Turn limit
- May be used in addition to Psychic Powers
- Willpower Checks for Psychic Gambits **cannot** inflict Perils of the Warp (unless specifically stated)

### Save Mechanics Implementation

```typescript
function isArmorNegated(weaponAP: number, armorSave: number): boolean {
  return weaponAP !== 0 && weaponAP <= armorSave
}

function getSaveFailProbability(
  armorSave: string,    // "3+"
  invulnSave?: string,  // "5++"
  ap: number            // 2
): number {
  const armorTarget = parseInt(armorSave)
  const invulnTarget = invulnSave ? parseInt(invulnSave) : 7

  // Check if armor is negated
  const armorNegated = (ap <= armorTarget)

  // Determine best save
  let bestSave = 7
  if (!armorNegated) {
    bestSave = Math.min(bestSave, armorTarget)
  }
  bestSave = Math.min(bestSave, invulnTarget)

  if (bestSave >= 7) {
    return 1.0  // No save available, 100% fail rate
  }

  const saveProb = (7 - bestSave) / 6
  return 1 - saveProb  // Return FAIL probability
}
```

### Monte Carlo Simulation

```typescript
function simulateCombat(scenario: CombatScenario, iterations: number = 10000): CombatResult {
  const results: number[] = []

  for (let i = 0; i < iterations; i++) {
    let totalWounds = 0

    for (let attack = 0; attack < scenario.attacks; attack++) {
      // Roll to hit
      const hitRoll = Math.random()
      if (hitRoll > scenario.hitProb) continue

      // Roll to wound
      const woundRoll = Math.random()
      if (woundRoll > scenario.woundProb) continue

      // Roll to save
      const saveRoll = Math.random()
      if (saveRoll > scenario.saveFailProb) continue

      // Apply damage
      totalWounds += scenario.damage
    }

    results.push(totalWounds)
  }

  return analyzeResults(results)
}
```

## Part 25: Edge Cases and FAQs

### Edge Case: S vs T at extremes

```
Q: Can S3 wound T9?
A: No. S3 vs T9 shows "-" on the wound chart, meaning impossible to wound without special rules like Poisoned.

Q: Can S10 wound T3?
A: Yes, on 2+. S10 vs T3 = 2+ to wound (see wound chart line for Strength 10).
```

### Edge Case: Invuln save with no armor save

```
Q: If AP negates armor, can I still take invuln?
A: Yes, invuln is independent of armor and ignores AP

Q: If I have no armor save at all (never had armor), can I still take invuln?
A: Yes, invulnerable saves work independently of armor saves. Models can have invuln without having armor.
```

### Edge Case: Critical hits and special rules

```
Q: If BS 7+ scores critical hit (auto-wound), do I still roll to wound for special rules like Rending?
A: No wound roll is made. Critical Hits automatically wound without rolling dice. However, the critical hit counts as rolling a '6' for purposes of any special rules that trigger on wound roll results. Note: Rending triggers on Hit Tests (not Wound Tests), so it's independent of critical hits.
```
